#### ğŸ”¹ What is a Function?

**Academic definition:**
A function is a block of code that performs a specific task and can be **called** multiple times.

**Beginner analogy:**
Think of a function like a blender: you put ingredients in, press a button, and get the smoothie. You can reuse it anytime without rewriting the steps.

**Python Syntax:**

```python
def function_name(parameters):
    """
    Optional documentation
    """
    # code block
    return value  # optional
```

---

#### ğŸ”¹ Example 1: Simple Function

```python

    print("Hello, welcome to Python!")
    
greet()
greet()  # can call multiple times
```

Output:

```
Hello, welcome to Python!
Hello, welcome to Python!
```

#### ğŸ”¹ Parameters and Arguments

Parameters allow functions to **receive input**.

```python
def greet_user(name):
    print(f"Hello, {name}!")

greet_user("Alice")
greet_user("Bob")
```

Output:

```
Hello, Alice!
Hello, Bob!
```

---

#### ğŸ”¹ Return Values

Functions can **process data and return a result**.

```python
def add(a, b):
    return a + b

result = add(5, 3)
print("Sum is:", result)
```

Output:

```
Sum is: 8
```

---

#### ğŸ”¹ Local vs Global Variables

* **Local variable:** Exists inside function, disappears after function ends
* **Global variable:** Declared outside, accessible everywhere

```python
x = 10  # global

def demo():
    y = 5  # local
    print("Inside function:", x, y)

demo()
print("Outside function, x =", x)
# y is not accessible here â†’ error
```

> ğŸ’¡ Best practice: minimize global variables. Functions should be **self-contained**.

---

#### ğŸ”¹ Why Functions?

* Reusability
* Readability
* Debugging easier
* Avoid repetition

**Example: Temperature conversion**

```python
def celsius_to_fahrenheit(c):
    return (c * 9/5) + 32

temps = [0, 20, 37, 100]

for t in temps:
    print(celsius_to_fahrenheit(t))
```
