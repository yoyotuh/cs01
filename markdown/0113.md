### ðŸ“˜ Deep Concepts Explained

#### ðŸ”¹ Why Use Functions?

* Functions **modularize code** â†’ easier to read, test, reuse
* Avoid **code repetition** â†’ DRY principle (Don't Repeat Yourself)
* Allow logical separation â†’ main program remains clean

**Real-world analogy:**

* Function = machine in factory â†’ input raw material, process, output product

#### ðŸ”¹ Parameters vs Arguments

* **Parameter:** variable in function definition
* **Argument:** actual value passed when calling

```python

    print(f"Hello, {name}!")

greet("Alice")    # "Alice" = argument
```

#### ðŸ”¹ Recursion Introduction

**Definition:** A function that calls itself to solve smaller instances of the same problem

**Important Rules:**

1. Must have **base case** â†’ stop recursion
2. Must **move toward base case**

**Example: Factorial**

```python
def factorial(n):
    if n == 0:
        return 1  # base case
    else:
        return n * factorial(n-1)  # recursive call

print(factorial(5))  # 120
```

**Visual intuition:**

* factorial(5) â†’ 5 * factorial(4) â†’ 5*4*factorial(3) â†’ ... â†’ 1

---

#### ðŸ”¹ When to Use Recursion

* Solving **divide-and-conquer problems**
* Tree/graph traversal
* Problems naturally defined recursively (factorial, Fibonacci)

**Warning:** Too deep recursion â†’ stack overflow
